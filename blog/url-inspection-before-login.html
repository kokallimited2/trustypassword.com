🔗 How to Inspect a URL Before Entering Your Password

By Sophie Laurent, Cybersecurity Awareness Trainer, Trusty Password · 21 Apr 2026 · 3 min read · 519 words

On this page

  • Understand the Structure of a URL
  • Find the Root Domain
  • Common URL Deception Techniques
  • How to Verify a URL on Any Device
  • Using Password Manager Autofill as a URL Check

Inspecting a URL before entering your password is the most important technical skill for avoiding phishing. A fake login page can look identical to the real one, but the URL in the address bar always tells the truth. Here is how to read a URL like a security professional.

Understand the Structure of a URL

Every URL has the same structure. Learn to identify each part:

https://www.example.com/login?return=home

  • https:// — The protocol (HTTP vs HTTPS). HTTPS is expected; HTTP on a login page is an immediate red flag.
  • www.example.com — The full domain name. This is what you must verify.
  • /login — The path. This matters less than the domain — attackers can put any path on any domain.
  • ?return=home — Query parameters. These should not be used to judge legitimacy.

The domain is the only part that identifies the website owner. Everything after the first forward slash belongs to that domain. An attacker cannot put "paypal.com" in the domain of a page they do not own.

Find the Root Domain

To find the root domain, read the domain from right to left. The top-level domain (TLD) is the last part (.com, .org, .co.uk). One step left is the root domain. Anything to the left of that is a subdomain — and can be anything.

Example: In login.paypal.com.security-alerts.xyz:

  • TLD: .xyz
  • Root domain: security-alerts.xyz
  • Subdomain: login.paypal.com (the attacker owns security-alerts.xyz and added login.paypal.com as a subdomain)

The real PayPal domain is paypal.com. Seeing "paypal" anywhere in a URL that does not end with paypal.com means you are on a different website.

Common URL Deception Techniques

Subdomain Spoofing

Attackers create subdomains that look like legitimate domains: amazon.com.au.redirect-verify.tk. The real domain is redirect-verify.tk, not amazon.com.au.

Typosquatting

Domains that differ by one character: amaz0n.com (digit zero instead of letter o), go0gle.com, micr0soft.com. Always verify exact spelling.

Trustworthy Path, Suspicious Domain

A phishing page might have /paypal/login/ in the URL path, but if the domain before it is wrong, the path is irrelevant. The path is just a folder on the attacker's server.

How to Verify a URL on Any Device

On Desktop

  1. Click the address bar to see the full URL (not truncated).
  2. Read the domain from right to left to find the root domain.
  3. Compare the root domain to the official domain of the organisation.
  4. If they do not match exactly, do not enter your password.

On Mobile

  1. Tap the address bar to expand it.
  2. Read the full domain carefully — mobile browsers often truncate long domains.
  3. If the domain looks suspicious, navigate directly by typing the known address into a new tab.

Using Password Manager Autofill as a URL Check

Password managers only autofill on the exact domain where credentials were saved. If you visit a page and your password manager does not offer to fill, the domain does not match — even if the page looks identical. This is the simplest and most reliable URL check available. If your password manager does not offer credentials, the URL is wrong.
